# n8n 워크플로우 구성 및 데이터 흐름 제어\n\n환영합니다! n8n을 활용한 자동화의 세계에 오신 것을 진심으로 환영합니다. 이 챕터에서는 n8n 워크플로우의 심장부에 해당하는 구성 요소와 데이터가 워크플로우를 따라 어떻게 흐르는지 깊이 있게 이해하는 것을 목표로 합니다. 특히, 워크플로우의 시작점인 트리거 노드와 데이터를 가공하고 처리하는 일반 노드의 차이점을 명확히 이해하고, 데이터를 효과적으로 조작하며, 조건부 로직과 반복문을 구현하는 방법을 다룰 것입니다. Python 프로그래밍 경험이 있는 분들이라면, n8n의 비주얼 인터페이스에서 이 개념들을 어떻게 적용하고 확장할 수 있는지에 대한 통찰을 얻게 될 것입니다.\n\n우리는 n8n의 기본기를 탄탄히 다져, 복잡한 자동화 워크플로우를 구축하는 데 필요한 기반 지식을 제공할 것입니다.\n\n## 1. n8n 워크플로우의 기본 개념\n\nn8n은 다양한 서비스와 애플리케이션을 연결하여 자동화된 워크플로우를 만들 수 있게 해주는 강력한 오픈 소스 자동화 도구입니다. 복잡한 코딩 없이 드래그 앤 드롭 인터페이스를 통해 시스템 간의 데이터 흐름을 시각적으로 설계할 수 있다는 것이 큰 장점입니다.\n\n*   **워크플로우(Workflow)란 무엇인가?**\n    워크플로우는 일련의 자동화된 작업을 의미합니다. 특정 이벤트(트리거)에 반응하여 시작되며, 여러 단계를 거쳐 데이터를 처리하고 특정 목표를 달성합니다. 예를 들어, 새로운 이메일이 오면 자동으로 특정 데이터를 추출하여 데이터베이스에 저장하고, 슬랙(Slack)으로 알림을 보내는 과정 전체가 하나의 워크플로우가 될 수 있습니다.\n\n*   **노드(Node)의 역할**\n    각 워크플로우는 여러 '노드'로 구성됩니다. 노드는 워크플로우의 개별 단계를 나타내며, 특정 작업을 수행하는 작은 블록입니다. 데이터를 받아서 처리하고, 다른 노드로 전달하는 역할을 합니다. n8n은 수백 가지의 내장 노드를 제공하여 다양한 서비스와의 통합 및 데이터 처리를 가능하게 합니다.\n\n*   **워크플로우 실행 방식**\n    n8n 워크플로우는 다음과 같은 다양한 방식으로 실행될 수 있습니다:\n    *   **수동 실행 (Manual Execution)**: 워크플로우 에디터에서 직접 'Execute Workflow' 버튼을 눌러 실행합니다.\n    *   **스케줄 실행 (Scheduled Execution)**: 특정 시간 간격(예: 매시간, 매일 자정)으로 자동 실행되도록 설정합니다.\n    *   **웹훅 실행 (Webhook Trigger)**: 외부 시스템에서 특정 URL로 HTTP 요청(웹훅)을 보낼 때 워크플로우가 시작됩니다.\n    *   **특정 이벤트 트리거**: 파일 시스템의 변경, 데이터베이스 업데이트 등 특정 이벤트가 발생할 때 시작되도록 설정할 수 있습니다.\n\n## 2. 트리거 노드와 일반 노드\n\nn8n 워크플로우는 항상 '시작점'이 필요합니다. 이 시작점을 정의하는 것이 바로 트리거 노드이며, 이후의 모든 데이터 처리와 변환은 일반 노드를 통해 이루어집니다.\n\n### 2.1. 트리거 노드 (Trigger Node)\n\n트리거 노드는 워크플로우의 **시작점**입니다. 특정 조건이나 이벤트가 발생했을 때 워크플로우가 실행되도록 하는 역할을 합니다. 트리거 노드는 일반적으로 외부 시스템의 이벤트를 '감지'하거나, 정기적으로 작업을 '스케줄링'하는 방식으로 작동합니다.\n\n*   **정의 및 역할**: 워크플로우 실행을 시작하는 유일한 노드입니다. 워크플로우당 단 하나의 트리거 노드만 존재할 수 있으며, 이 노드는 외부로부터 데이터를 받거나 특정 시간/이벤트에 따라 내부적으로 데이터를 생성하여 다음 노드로 전달합니다.\n\n*   **주요 트리거 노드 유형**: n8n은 다양한 유형의 트리거 노드를 제공합니다.\n    *   **Webhook**: 외부 애플리케이션이나 서비스가 특정 URL로 HTTP 요청(GET, POST 등)을 보낼 때 워크플로우를 실행합니다. 가장 흔하게 사용되는 트리거 중 하나입니다. (예: `opal.google`에서 특정 이벤트 발생 시 n8n으로 데이터 전송)\n    *   **Schedule**: 지정된 시간 간격(예: 매 5분, 매주 월요일)으로 워크플로우를 실행합니다.\n    *   **CRON**: CRON 표현식을 사용하여 더욱 세밀한 스케줄링을 설정할 수 있습니다.\n    *   **On start**: n8n 인스턴스가 시작될 때 한 번만 워크플로우를 실행합니다.\n    *   **Watch Files**: 특정 디렉토리의 파일 변경(생성, 수정, 삭제)을 감지하여 워크플로우를 실행합니다.\n\n*   **예시: Webhook 트리거 설정**\n    가장 간단한 웹훅 트리거는 다음과 같이 설정할 수 있습니다.\n    1.  새 워크플로우를 생성합니다.\n    2.  `Webhook` 노드를 검색하여 추가합니다.\n    3.  노드 설정에서 'HTTP Method'를 `POST`로, 'Response Mode'를 `On “Webhook Response” Node`로 설정합니다.\n    4.  'Webhook URL'을 복사합니다. 이 URL로 데이터를 `POST`하면 워크플로우가 실행됩니다.\n    5.  'Test Webhook'을 클릭하여 테스트 모드를 활성화한 후, 복사한 URL로 Postman과 같은 도구를 사용하여 테스트 요청을 보냅니다.\n\n### 2.2. 일반 노드 (Regular Node)\n\n일반 노드는 트리거 노드로부터 전달받은 데이터를 **처리하고 변환**하는 역할을 합니다. 워크플로우의 핵심 로직을 구성하며, 다양한 작업을 수행할 수 있습니다.\n\n*   **정의 및 역할**: 트리거 노드 이후에 연결되어 데이터를 조작, 필터링, 전송하거나 다른 서비스와 상호작용합니다. 각 일반 노드는 이전 노드로부터 데이터를 입력받아 특정 작업을 수행하고, 그 결과를 다음 노드로 출력합니다.\n\n*   **주요 일반 노드 유형**: n8n에는 수많은 일반 노드가 있으며, 몇 가지 주요 유형은 다음과 같습니다.\n    *   **Set**: 데이터를 생성하거나 기존 데이터를 수정합니다.\n    *   **Code**: JavaScript/TypeScript 코드를 사용하여 복잡한 로직을 구현합니다. (Python 경험자들이 자신의 로직을 적용하기 좋은 노드)\n    *   **HTTP Request**: 외부 API에 HTTP 요청을 보냅니다.\n    *   **If**: 조건에 따라 워크플로우의 흐름을 분기합니다.\n    *   **Split In Batches**: 여러 항목으로 구성된 데이터를 작은 배치로 나누어 개별적으로 처리합니다.\n    *   **Google Sheets, Slack, Email 등 서비스별 노드**: 특정 서비스와 상호작용하는 미리 정의된 노드들입니다.\n\n*   **트리거 노드와의 차이점**: 가장 큰 차이점은 역할입니다. 트리거 노드는 워크플로우를 '시작'하는 반면, 일반 노드는 시작된 워크플로우 내에서 데이터를 '처리'합니다. 일반 노드는 항상 이전 노드로부터 데이터를 입력받아야만 작동할 수 있습니다. (예외적으로 `Start` 노드는 데이터를 생성할 수 있지만, 일반적인 의미의 트리거 노드는 아님)\n\n## 3. n8n에서의 데이터 흐름 이해\n\nn8n 워크플로우의 핵심은 데이터가 노드와 노드 사이를 어떻게 이동하고 변환되는지 이해하는 것입니다. 모든 데이터는 JSON 객체 형태로 전달됩니다.\n\n### 3.1. 데이터 생성 및 구조\n\n각 노드는 이전 노드에서 전달받은 데이터를 입력으로 사용하고, 처리 결과물을 JSON 객체 형태로 다음 노드로 출력합니다. 이 출력 데이터는 `json`이라는 최상위 키 안에 여러 아이템(items)의 배열 형태로 저장됩니다.\n\n*   **기본 데이터 구조**: `[{ \"json\": { \"key1\": \"value1\", \"key2\": \"value2\" } }]`\n*   **예시: `Set` 노드를 이용한 데이터 생성**\n    `Set` 노드를 사용하여 다음과 같은 데이터를 생성할 수 있습니다.\n    1.  `Set` 노드를 추가합니다.\n    2.  'Properties' 섹션에서 'Add Value'를 클릭하고 'String'을 선택합니다.\n    3.  'Name'을 `userName`, 'Value'를 `김앤N`으로 설정합니다.\n    4.  'Add Value'를 다시 클릭하고 'Number'를 선택합니다.\n    5.  'Name'을 `userAge`, 'Value'를 `30`으로 설정합니다.\n    6.  노드를 실행하면 다음과 같은 출력을 볼 수 있습니다.\n    \n    [\n      {\n        \"json\": {\n          \"userName\": \"김앤N\",\n          \"userAge\": 30\n        }\n      }\n    ]\n    \n\n### 3.2. 데이터 변환 및 접근\n\n이전 노드의 데이터에 접근하고 변환하는 것은 n8n 워크플로우에서 매우 중요합니다. n8n은 '표현식(Expressions)'을 사용하여 이전 노드의 데이터에 접근할 수 있도록 합니다.\n\n*   **표현식 사용법 (Expressions)**: `{{ }}` 중괄호 안에 JavaScript 문법을 사용하여 이전 노드의 데이터에 접근합니다.\n    *   `{{ $json.keyName }}`: 현재 아이템의 `json` 객체 내의 `keyName` 값에 접근합니다.\n    *   `{{ $node[\"NodeName\"].json.keyName }}`: 특정 노드(`NodeName`)의 출력 데이터 중 `json` 객체 내의 `keyName` 값에 접근합니다.\n    *   `{{ $item(0).$json.keyName }}`: 첫 번째 아이템의 `json` 객체 내 `keyName` 값에 접근합니다.\n    *   `{{ $parameter.parameterName }}`: 현재 노드의 파라미터 값에 접근합니다.\n\n*   **예시: `Set` 노드에서 이전 노드의 데이터 참조**\n    `Webhook` 노드로부터 `{\"name\": \"철수\", \"city\": \"서울\"}` 데이터를 받았다고 가정해 봅시다. 이 데이터를 가공하여 인사말을 만드는 워크플로우입니다.\n    1.  `Webhook` 노드를 추가하고 테스트 데이터로 `{\"name\": \"철수\", \"city\": \"서울\"}`을 받습니다.\n    2.  `Set` 노드를 추가합니다.\n    3.  `Set` 노드에서 'Name'을 `greeting`, 'Value'를 `안녕하세요, {{ $json.name }}님. {{ $json.city }}에 오신 것을 환영합니다!`로 설정합니다.\n    4.  결과는 `{\"greeting\": \"안녕하세요, 철수님. 서울에 오신 것을 환영합니다!\"}` 가 됩니다.\n\n### 3.3. 다음 노드로의 데이터 전달\n\n노드들은 시각적으로 화살표로 연결되어 데이터의 흐름을 보여줍니다. 데이터는 이 연결선을 따라 한 노드에서 다음 노드로 자동으로 전달됩니다.\n\n*   **노드 간의 연결 방식**: 노드의 출력 포트(오른쪽)를 다음 노드의 입력 포트(왼쪽)로 드래그하여 연결합니다.\n*   **여러 출력(output)을 가진 노드의 경우**: `If` 노드와 같이 조건에 따라 다른 경로로 데이터를 보낼 수 있는 노드는 여러 개의 출력 포트를 가집니다. 각 포트는 특정 조건이 만족되었을 때만 데이터를 전달합니다.\n\n## 4. 조건부 로직 구현 (If 노드)\n\n자동화 워크플로우에서는 특정 조건에 따라 다른 작업을 수행해야 하는 경우가 많습니다. n8n에서는 `If` 노드를 사용하여 이러한 조건부 로직을 쉽게 구현할 수 있습니다.\n\n### 4.1. `If` 노드\n\n`If` 노드는 이전 노드에서 전달받은 데이터가 특정 조건을 만족하는지에 따라 워크플로우의 흐름을 분기합니다. 'True'와 'False' 두 개의 출력 포트를 가지며, 조건이 참이면 'True' 포트로, 거짓이면 'False' 포트로 데이터를 보냅니다.\n\n*   **개념 및 사용 목적**: 특정 데이터 값이 특정 기준을 충족하는지 확인하고, 그 결과에 따라 후속 작업을 다르게 하고자 할 때 사용합니다. (예: 구매 금액이 10만원 이상이면 VIP 고객 처리, 아니면 일반 고객 처리)\n\n*   **설정 방법**: `If` 노드 설정에서는 비교할 값, 연산자(equal, greater than, contains 등), 그리고 비교 대상 값을 지정합니다. 여러 조건을 `AND` 또는 `OR`로 연결하여 복합적인 조건을 만들 수도 있습니다.\n\n*   **예시 워크플로우: 조건에 따라 다른 이메일 발송**\n    1.  `Webhook` 노드로 사용자 데이터(`{\"email\": \"user@example.com\", \"country\": \"Korea\"}`)를 받습니다.\n    2.  `If` 노드를 추가하고 다음과 같이 설정합니다.\n        *   `Value 1`: `{{ $json.country }}`\n        *   `Operation`: `Is Equal`\n        *   `Value 2`: `Korea`\n    3.  `If` 노드의 'True' 출력 포트에 `Email Send` 노드를 연결하고, 'False' 출력 포트에도 다른 `Email Send` 노드를 연결합니다.\n    4.  'True' 측의 `Email Send` 노드는 '제목: 국내 고객 환영', 'False' 측의 `Email Send` 노드는 '제목: 해외 고객 환영'으로 설정합니다. 이렇게 하면 'country' 값이 'Korea'인 경우와 아닌 경우에 따라 다른 이메일이 발송됩니다.\n\n## 5. 반복문 구현 (Split In Batches, Code 노드)\n\n데이터가 여러 항목으로 구성된 리스트(배열)일 때, 각 항목에 대해 동일한 작업을 반복해야 하는 경우가 있습니다. n8n에서는 `Split In Batches` 노드와 `Code` 노드를 활용하여 반복문을 구현할 수 있습니다.\n\n### 5.1. `Split In Batches` 노드\n\n이 노드는 여러 개의 아이템을 작은 묶음(배치)으로 나누어 처리할 때 유용합니다. 특히 대량의 데이터를 한 번에 처리하는 대신, 일정량씩 나누어 처리하여 시스템 부하를 줄이거나 API 호출 제한을 준수할 때 사용됩니다.\n\n*   **개념 및 사용 목적**: 입력으로 받은 아이템 리스트를 지정된 배치(Batch) 크기만큼 분할하여 다음 노드로 순차적으로 전달합니다. 예를 들어, 1000개의 사용자 목록이 있을 때, `Split In Batches`를 사용하여 100개씩 10번 워크플로우를 실행하도록 만들 수 있습니다.\n\n*   **설정 방법**: 'Batch Size'를 설정하여 한 번에 처리할 아이템의 개수를 지정합니다.\n\n*   **예시: 대량의 사용자 목록을 개별 처리**\n    1.  `Set` 노드로 사용자 목록 배열을 생성합니다. (예: `[{\"name\": \"A\"}, {\"name\": \"B\"}, {\"name\": \"C\"}, ...]`) 이 노드는 여러 개의 아이템을 출력합니다.\n    2.  `Split In Batches` 노드를 추가하고 'Batch Size'를 `1`로 설정합니다. (각 사용자 개별 처리)\n    3.  `Split In Batches` 노드의 다음으로 각 사용자에 대해 특정 작업을 수행하는 노드(예: `HTTP Request`로 개별 사용자 정보 업데이트)를 연결합니다.\n\n### 5.2. `Code` 노드를 활용한 고급 반복/조건 (Python 경험 활용)\n\n`Code` 노드는 JavaScript/TypeScript 코드를 직접 작성하여 n8n의 기본 노드로는 구현하기 어려운 복잡한 로직을 처리할 수 있게 해줍니다. Python 프로그래밍에 익숙한 사용자라면, `for` 루프나 복잡한 `if/else` 문과 같은 개념을 이 `Code` 노드에 적용하여 n8n 워크플로우의 강력함을 극대화할 수 있습니다.\n\n*   **Python 경험자를 위한 팁**: n8n의 `Code` 노드는 JavaScript/TypeScript 문법을 사용하지만, `for` 루프, `if/else` 조건문, 변수 선언 및 함수 호출 등 기본적인 프로그래밍 개념은 Python과 유사합니다. Python에서 리스트를 반복 처리하듯이, `Code` 노드에서는 `items` 배열을 반복하여 각 아이템에 접근할 수 있습니다.\n\n*   **`Code` 노드 내에서 `for` 루프 사용 예시**: 이전 노드로부터 받은 모든 아이템을 반복하면서 특정 필드를 수정하는 경우.\n    javascript\n    // 입력 데이터를 가져옵니다. 각 아이템은 $json 속성 내에 데이터를 가집니다.\n    for (const item of items) {\n      // 각 아이템의 JSON 데이터에 접근합니다.\n      const userData = item.json;\n\n      // 조건에 따라 데이터를 수정하거나 새 속성을 추가합니다.\n      if (userData.age >= 18) {\n        userData.status = '성인';\n      } else {\n        userData.status = '미성년자';\n      }\n\n      // 이 아이템을 출력으로 반환합니다. (선택 사항, 기본적으로 입력 아이템이 반환됨)\n      // return item;  // 필요에 따라 수정된 item을 반환할 수 있습니다.\n    }\n    // 만약 새로운 아이템 배열을 생성하려면, 다음과 같이 반환할 수 있습니다.\n    // return items;\n    \n\n*   **`Code` 노드 내에서 `if/else` 문 사용 예시**: 특정 조건에 따라 다른 데이터 구조를 생성하거나 다른 로직을 실행하는 경우.\n    javascript\n    // 첫 번째 아이템의 데이터를 가져옵니다.\n    const inputData = items[0].json;\n    const outputItems = [];\n\n    if (inputData.category === 'premium') {\n      outputItems.push({\n        json: {\n          productName: inputData.name + ' (Premium)',\n          price: inputData.price * 1.2\n        }\n      });\n    } else {\n      outputItems.push({\n        json: {\n          productName: inputData.name,\n          price: inputData.price\n        }\n      });\n    }\n\n    return outputItems;\n    \n\n*   **`$json` 및 `items` 객체에 접근하는 방법**: `Code` 노드 내에서는 전역 변수 `items`를 통해 이전 노드에서 전달받은 모든 아이템 배열에 접근할 수 있습니다. 각 `item` 객체는 `json` 속성을 통해 실제 데이터에 접근할 수 있습니다.\n\n## 6. 실습 예제: 통합 워크플로우 구축\n\n이제까지 배운 내용을 바탕으로 간단한 통합 워크플로우를 만들어 봅시다. 웹훅으로 사용자 데이터가 들어오면, 특정 조건에 따라 데이터를 변환하고, 조건에 따라 다른 메시지를 보내는 시나리오입니다.\n\n1.  **트리거 노드**: `Webhook` 노드를 추가합니다. HTTP Method는 `POST`로 설정하고 URL을 복사합니다.\n2.  **데이터 초기화/추가**: `Set` 노드를 추가하고 `Webhook` 노드의 출력(`{{ $json }}`)을 기반으로 새로운 필드를 추가합니다. 예를 들어, `{{ $json.name }}`과 `{{ $json.score }}`를 받아서 `grade` 필드를 추가합니다.\n    *   `Value 1`: `grade` (Name)\n    *   `Value 2`: `{{ $json.score > 90 ? 'A' : ($json.score > 70 ? 'B' : 'C') }}` (Value - 삼항 연산자를 이용한 조건부 로직)\n3.  **조건 분기**: `If` 노드를 추가합니다.\n    *   `Value 1`: `{{ $json.grade }}` (Set 노드에서 생성된 `grade` 필드)\n    *   `Operation`: `Is Equal`\n    *   `Value 2`: `A`\n4.  **조건에 따른 처리 (True)**: `If` 노드의 'True' 포트에 `Email Send` 노드를 연결합니다. '제목: 성적 우수자'로 설정하고, '본문: {{ $json.name }}님, 훌륭한 성적입니다!' 와 같이 작성합니다.\n5.  **조건에 따른 처리 (False)**: `If` 노드의 'False' 포트에 다른 `Email Send` 노드를 연결합니다. '제목: 추가 학습 필요'로 설정하고, '본문: {{ $json.name }}님, 더 노력해 봅시다.' 와 같이 작성합니다.\n6.  **테스트**: Postman 또는 `curl`을 사용하여 `Webhook` URL로 JSON 데이터를 `POST`합니다. (예: `{\"name\": \"존\", \"score\": 95}` 또는 `{\"name\": \"제인\", \"score\": 70}`). 결과적으로 두 이메일 중 하나가 발송되는 것을 확인할 수 있습니다.\n\n## 7. 요약 및 다음 챕터 예고\n\n이 챕터에서는 n8n 워크플로우의 기본적인 구성 요소와 데이터 흐름을 깊이 있게 이해하는 시간을 가졌습니다. 트리거 노드와 일반 노드의 역할과 차이점을 명확히 했고, JSON 형태의 데이터가 노드 사이를 어떻게 이동하고 변환되는지 학습했습니다. 또한, `If` 노드를 사용한 조건부 로직과 `Split In Batches`, 그리고 `Code` 노드를 활용한 반복문 구현 방법을 익혔습니다. Python 경험이 있는 학습자분들은 `Code` 노드가 기존 프로그래밍 지식을 n8n 환경에서 활용할 수 있는 강력한 도구임을 이해하셨을 것입니다.\n\n이제 여러분은 n8n 워크플로우를 구성하고 기본적인 데이터 흐름을 제어할 수 있는 준비를 마쳤습니다. 다음 챕터에서는 **`opal.google`을 활용하여 워크플로우를 설계하고, 이를 n8n 워크플로우로 번역하는 방법**에 대해 자세히 다룰 예정입니다. 복잡한 로직을 `opal.google`에서 시각적으로 구상한 후, n8n의 비주얼 인터페이스로 옮겨오는 과정을 학습하며, 더욱 실용적인 자동화 기술을 습득하게 될 것입니다.
